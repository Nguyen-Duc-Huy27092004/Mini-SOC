## 🐛 Critical Bugs Fixed

### 1. ✅ **Migration Chain Fixed** (Already resolved in previous work)

**Problem:**
- Migration 004 had wrong `down_revision` reference
- Migration 004 tried to create indexes on columns not yet existing
- Migration chain: `001 → 002 → 003_incidents_workflow → 004` (BROKEN)

**Fix:**
- Changed 004 `down_revision` from `'003_incidents_workflow'` to `'003_incidents'`
- Moved indexes that depend on new columns to migration 005
- Correct chain: `001 → 002 → 003 → 004 → 005` ✅

**Files Modified:**
- `backend/alembic/versions/004_performance_indexes.py`
- `backend/alembic/versions/005_fill_missing_columns.py`

---

### 2. ✅ **CSRF_VALIDATE_ORIGIN Missing** (Already resolved)

**Problem:**
- `backend/app/middleware/csrf.py` reads `settings.CSRF_VALIDATE_ORIGIN`
- But `backend/app/core/config.py` doesn't define this field
- Backend crashes on startup with `AttributeError`

**Fix:**
- Added `CSRF_VALIDATE_ORIGIN: bool = False` to Settings class
- Added to `.env.production` template in deploy script

**Files Modified:**
- `backend/app/core/config.py`

---

### 3. ✅ **AlertsFileTailer Initialization** (Already resolved)

**Problem:**
- `collector/service.py` tried to initialize AlertsFileTailer with wrong parameters
- Missing `CollectorConfig` import
- Would crash collector service immediately

**Fix:**
```python
# Before (BROKEN):
from app.collector.alerts_tail import AlertsFileTailer
self.tailer = AlertsFileTailer(alerts_file=self.alerts_file)

# After (FIXED):
from app.collector.alerts_tail import AlertsFileTailer, CollectorConfig
self.tailer = AlertsFileTailer(
    config=CollectorConfig(),
    alerts_file=self.alerts_file or None,
)
```

**Files Modified:**
- `backend/app/collector/service.py`

---

### 4. ✅ **Deploy Script: Migrations Run AFTER Backend Start** 🔴 CRITICAL

**Problem:**
- Original deploy script started ALL services first
- Then tried to run migrations
- Backend would crash trying to access non-existent database tables
- **This was the main cause of deployment failures**

**Fix - Multi-step deployment sequence:**
```bash
# NEW CORRECT ORDER:
1. Build all images
2. Start ONLY db + redis
3. Wait for db to be ready (up to 120s)
4. Run migrations using one-off container
5. Start backend + frontend + nginx
6. Verify health checks
7. Create admin user
```

**Key Changes:**
- Run `docker-compose run --rm backend alembic upgrade head` BEFORE starting backend service
- Added proper health checks with retries
- Added error logging for each step
- Exit immediately on failure with diagnostic logs

**Files Modified:**
- `deploy_on_wazuh.sh` (lines 402-530)

---

### 5. ✅ **Missing WAZUH_ALERTS_HOST_PATH Variable**

**Problem:**
- `docker-compose.production.yml` uses `${WAZUH_ALERTS_HOST_PATH:-./data/wazuh}`
- But `.env.production` generated by script doesn't define this variable
- Could cause Wazuh alerts mount to fail silently

**Fix:**
- Added `WAZUH_ALERTS_HOST_PATH` to `.env.production` generation
- Added script step to create directory with proper permissions
- Value: `$DEPLOY_PATH/data/wazuh`

**Files Modified:**
- `deploy_on_wazuh.sh` (lines 275-280, 402-405)

---

### 6. ✅ **Backend Dockerfile Missing PYTHONPATH**

**Problem:**
- Backend container working directory is `/app`
- But Python imports may fail without explicit PYTHONPATH
- `alembic` commands could fail with import errors

**Fix:**
- Added `ENV PYTHONPATH="/app"` to Dockerfile
- Added `netcat-openbsd` for network troubleshooting

**Files Modified:**
- `docker/backend.Dockerfile` (line 13)

---

### 7. ✅ **Create Admin Script Execution Path**

**Problem:**
- Deploy script runs: `python app/scripts/create_admin_user.py`
- But container working directory is `/app`
- Script may not be in expected location

**Fix:**
- Changed to use Python inline execution with proper module import
- Added validation for email format (regex)
- Added password length validation (min 8 chars)
- Added better error handling and fallback instructions

**Files Modified:**
- `deploy_on_wazuh.sh` (lines 550-580)

---

### 8. ✅ **Deploy Script: Insufficient Error Handling**

**Problem:**
- Original script used `|| true` to hide errors
- No timeout on service startup waits
- No diagnostic output on failure
- Silent failures would leave broken deployment

**Fix:**
- Added explicit error checking for every critical step
- Added retry loops with timeouts (60-120 seconds)
- Added diagnostic log output on failures
- Added build log capture to `/tmp/backend_build.log` and `/tmp/frontend_build.log`
- Exit immediately on failure instead of continuing

**Example:**
```bash
# Before:
docker-compose up -d
sleep 30

# After:
BACKEND_READY=false
for i in {1..60}; do
    if curl -sf http://localhost:8000/api/v1/health/ready; then
        BACKEND_READY=true
        break
    fi
    sleep 2
done

if [ "$BACKEND_READY" = false ]; then
    log_error "Backend failed to start. Showing logs:"
    docker-compose logs --tail=100 backend
    exit 1
fi
```

**Files Modified:**
- `deploy_on_wazuh.sh` (entire script refactored)

## 🔧 Rollback Procedure

If deployment fails:

```bash
# Stop all containers
cd /opt/mini-soc
docker-compose -f docker-compose.production.yml down -v

# Check logs
docker-compose -f docker-compose.production.yml logs backend > backend.log
docker-compose -f docker-compose.production.yml logs db > db.log

# Clean up
docker system prune -f

# Restore backup if exists
cp docker-compose.production.yml.bak docker-compose.production.yml

# Retry deployment
sudo bash deploy_on_wazuh.sh
```

## 📞 Troubleshooting

### Backend won't start:
```bash
# Check logs
docker-compose -f docker-compose.production.yml logs backend

# Common issues:
# - Database connection failed → Check POSTGRES_PASSWORD in .env.production
# - Redis connection failed → Check REDIS_PASSWORD
# - Import errors → Check PYTHONPATH in Dockerfile
```

### Migrations fail:
```bash
# Check database is accessible
docker-compose exec db psql -U postgres -d mini_soc_prod -c "SELECT 1;"

# Run migrations manually with debug
docker-compose run --rm backend alembic upgrade head --verbose

# Check migration files
docker-compose exec backend ls -la alembic/versions/
```

### Frontend 404 errors:
```bash
# Check nginx config
docker-compose exec nginx cat /etc/nginx/nginx.conf

# Check frontend build
docker-compose logs frontend

# Rebuild frontend
docker-compose build --no-cache frontend
docker-compose up -d frontend nginx
```

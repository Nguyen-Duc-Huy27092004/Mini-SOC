fix: authentication bugs - 403 refresh & infinite loop

Fixed 2 critical authentication bugs causing console errors:

BUG #8: Axios Interceptor Infinite Loop 🔴 CRITICAL
- Problem: /auth/me fails 401 → interceptor calls /auth/refresh
  → /auth/refresh fails → interceptor retries /auth/refresh again
  → infinite loop consuming resources
  
- Root cause: Interceptor was retrying ALL 401 responses including
  auth endpoints themselves
  
- Fix: Added isAuthEndpoint check to prevent retry on:
  * /auth/refresh (prevent loop)
  * /auth/login (prevent double login)
  
- Impact: No more infinite refresh loops, cleaner console output

BUG #9: /auth/refresh Returns 403 Instead of 401 🔴 CRITICAL  
- Problem: When cookies are stale/partial, /auth/refresh validates
  CSRF first and returns 403, but should return 401 for auth failures
  
- Root cause: validate_csrf() called before checking refresh token,
  causing confusing 403 Forbidden errors
  
- Fix: Reordered validation logic:
  1. Check refresh token exists → 401 if missing
  2. Try validate CSRF → catch exception
  3. If CSRF fails → clear cookies, return 401 (not 403)
  4. Continue with token refresh
  
- Impact: Consistent 401 responses for authentication failures,
  proper cookie cleanup, better UX
